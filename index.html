1. What is a Convolution?

A convolution is the method of combining two signals to generate a new output signal. More specifically, we can obtain an output signal \(g[n]\) by convolving an input signal \(x[n]\) and a kernel \(h[n]\).

Note: A kernel can sometimes be referred to as an impulse signal.
Note: All signals discussed on this webpage will be discrete unless stated otherwise.

Mathematically, the formula for discrete convolution is as follows: $$g[n] = x[n] * h[n]$$

Convolving a signal can have many useful applications, such as smoothing signals, filtering, and edge detection.

1.1. How do you perform a Convolution?

You can think of it as a small window, the kernel in this case, sliding across the input signal. At each timestep of the sliding window, we perform pairwise multiplication on the overlapping values and sum them to produce a new output value.

1.3. Observations

Notice that the kernel is "flipped" when sliding across the input signal. This comes directly from the definition of the convolution integral: $$(x * h)(t) = \int_{-\infty}^{\infty} x(\tau)\, h(t - \tau)\, d\tau$$

The term \((t - \tau)\) evaluates the kernel \(h\) in reversed time, which is equivalent to flipping the kernel before shifting it across the input signal.

In this demonstration, the effect on the output may appear subtle because the chosen kernel is relatively simple. We will explore kernels that are more expressive in the later sections.

2. Linear vs. Circular Convolutions

Linear convolution can be thought of as the true convolution between two signals, where one signal slides fully across the other without any boundary assumptions.

Circular convolution, on the other hand, treats signals as if they are periodic and hence, repeat indefinitely. When the kernel moves past the end of the signal, it wraps back around to the beginning, This behaviour results in overlaps that would not occur in linear convolution, and is usually not what we want when performing convolution.

2.1. Avoiding Circular Convolution

This distinction is important because the Discrete Fourier Transform (DFT) naturally computes circular convolution. To achieve linear convolution using DFT, we can pad both the input signal and kernel with zeroes to avoid the wrap-around effect.

  • In the case of 1D, if the input signal has length \(m\) and the kernel has length \(p\), both must be zero-padded to a minimum length of \((m + p − 1)\) before performing the DFT.

  • In the case of 2D, if the input signal is of size \(m\) by \(n\) and the kernel is of size \(p\) by \(q\), both must be zero-padded to a minimum size of \((m + p − 1)\) by \((n + q − 1)\) before performing the DFT.

3. Relationship between Convolution and Pairwise Multiplication

Convolution and pairwise multiplication are two different ways of describing the same operation in different domains. In other words, performing convolution of two signals in the time domain is equivalent to performing pairwise multiplication in the frequency domain.

Note: The mathematical proof behind the above assertion is not important for our purposes. See Convolution Theorem for more information.

This duality allows us to switch between domains depending on which operation is more convenient. In particular, convolution in the time domain can be computed more efficiently by transforming signals into the frequency domain, performing a simple multiplication, and then transforming the result back.

The operation described can be formally notated as:

\[ g[n] = \mathcal{F}^{-1} \{ H[k] \cdot X[k] \} \]

where \(H[k] = \mathcal{F}\{h[n]\}\) and \(X[k] = \mathcal{F}\{x[n]\}\), with \(\mathcal{F}\) denoting the Discrete Fourier Transform (DFT).

4. Applications of Convolution

Convolution is a fundamental operation used across domains such as:

  • Processing audio signals
  • Processing/analysing images

While the underlying mathematical principle remains the same, its applications can vary depending on whether the data is one-dimensional or two-dimensional.

4.1. One-Dimensional (1D) Applications of Convolution

In one-dimensional signals such as audio or time-series data, convolution can be used to modify or analyse how a signal changes over time. By applying different kernels, we can smooth out noise, remove unwanted frequency components, or simulate real-world effects such as echoes and reverberation.

These operations can be performed either directly in the time domain using convolution, or more efficiently in the frequency domain using pairwise multiplication via the Fourier Transform. A few applications of 1D convolution can be seen below:

Application Purpose Operation performed in
Time Domain Frequency Domain
Signal Smoothing Reduce noise and rapid fluctuations in the signal Convolution Pairwise Multiplication
Low Pass Filter Remove high-frequency components Convolution Pairwise Multiplication
Adding Audio Effects (e.g. echo, reverb) Modify signal using impulse response Convolution Pairwise Multiplication

4.2. Two-Dimensional (2D) Applications of Convolution

In two-dimensional signals such as images, convolution operates over both height and width, allowing us to process spatial features. By applying different kernels, we can blur images, sharpen details, detect edges, or extract meaningful patterns.

Similar to the 1D case, these operations can be performed in the spatial domain using convolution, or in the frequency domain using pairwise multiplication with the 2D Fourier Transform. A few applications of 2D convolution can be seen below:

Application Purpose Operation performed in
Spatial Domain Frequency Domain
Image Smoothing (Blurring) Reduce noise and fine detail in an image Convolution (e.g. Gaussian kernel) Pairwise Multiplication
Image Sharpening Enhance edges and fine details Convolution Pairwise Multiplication
Edge Detection Highlight boundaries and object edges Convolution (e.g. Sobel, Laplacian kernels) Pairwise Multiplication
Feature Extraction Detect patterns such as textures or shapes Convolution (learned filters, e.g. CNNs) Pairwise Multiplication

5. Conclusion

Convolution is a powerful and versatile operation that allows us to combine signals and extract meaningful information from them. Whether applied to one-dimensional signals such as audio or two-dimensional data such as images, the idea of the sliding kernel remains the same.

Beyond its intuitive interpretation in the time/spatial domain, convolution shows a connection to the frequency domain, where it becomes simple pairwise multiplication. This relationship allows for more efficient computation and provides valuable insight into how signals behave under different transformations.
